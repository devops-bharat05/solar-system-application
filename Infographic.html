Application Workflow

The application uses a standard client-server architecture. The user's browser interacts with a Node.js backend, which in turn communicates with the MongoDB database to fetch and serve planet data.

Technical Deep-Dive

API Endpoints

GET /api/planets

Retrieves a list of all planets, with optional start/end query params.

GET /api/planets/search

Searches for a planet by its name.

GET /api/podname

Returns the K8s pod name or a fallback message.
